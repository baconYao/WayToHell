# 測試資料集 (tests)

本目錄存放 Big2 遊戲的**端對端測試資料**，每組測資由一對檔案組成：

- **`<名稱>.in`**：標準輸入（洗好的牌堆、四位玩家名稱、依序的玩家動作）
- **`<名稱>.out`**：對應的預期標準輸出

程式行為應為：以 `.in` 作為 stdin 執行後，stdout 需與 `.out` 完全一致。

---

## 測資一覽

| 檔名                       | 用途                                                                                                                                                                                         |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **normal-no-error-play1**  | 與專案 Readme 範例相同。正常流程、無非法輸入：單張、對子、PASS、多回合，直到一方獲勝。用於驗證基本流程與輸出格式。                                                                           |
| **normal-no-error-play2**  | 另一組正常流程，以**對子**為主。驗證對子牌型辨識、大小比較與回合輪替。                                                                                                                       |
| **illegal-actions**        | 驗證**非法動作**的處理：新回合首輪不得 PASS（應輸出「你不能在新的回合中喊 PASS」）、不合法牌型（應輸出「此牌型不合法，請再嘗試一次。」），程式需拒絕並重新輪到同一玩家直到給出合法動作為止。 |
| **always-play-first-card** | 牌堆為依序排列（每人同花色 3~2），每輪每位玩家都出手中索引 0 的牌。用於驗證發牌順序、回合順序與「總是出最小牌」的簡單策略輸出。                                                              |
| **straight**               | 驗證**順子**牌型：包含打出順子（如 C[3] C[Q] S[K] D[A] H[2]）、順子與其他牌型的輪替、以及順子的大小比較。                                                                                    |
| **fullhouse**              | 驗證**葫蘆**牌型：包含打出葫蘆（如 C[3] D[3] H[3] H[6] S[6]）、葫蘆與單張等牌型的輪替、以及葫蘆的大小比較。                                                                                  |

---

## 使用方式

可依需求用下列方式跑測資（需先實作完整遊戲邏輯）：

```bash
# 單一測資
go run ./cmd/api < tests/normal-no-error-play1.in | diff - tests/normal-no-error-play1.out

# 或編譯後
./api < tests/normal-no-error-play1.in | diff - tests/normal-no-error-play1.out
```

若 `diff` 無輸出則表示通過。亦可寫成腳本或 Makefile target 批次跑完所有 `.in` / `.out` 配對。
